KubePi [kubəˈpaɪ]

现代化的 Kubernetes 多集群管理面板，帮助管理员统一接入集群、细化集群与命名空间权限，并让开发人员更高效地管理应用和定位问题。

一条命令启动 KubePi

运行 Docker 镜像即可完成体验。登录后导入 Kubernetes 集群，就可以开始管理资源。

也可通过图形化方式部署：前往 1Panel 应用商店

Docker 
docker run --privileged -d --restart=unless-stopped -p 80:80 1panel/kubepi
默认用户：admin 默认密码：kubepi

一个面向团队协作的 K8s 控制台

KubePi 将集群管理、资源操作、权限划分和排障入口收敛到统一的 Web 界面，让 Kubernetes 的日常使用更直观。

管理员

接入多个 Kubernetes 集群，围绕团队、集群和命名空间建立清晰的访问边界。

开发人员

在被授权的范围内查看资源、管理应用、排查服务运行状态，减少频繁切换命令行上下文。

运维团队

结合监控、指标和日志审计能力，把集群资源状态与日常运维动作放在同一个入口。

保留 Kubernetes 的完整能力，降低日常使用门槛

KubePi 覆盖多集群接入、资源管理、权限控制与运维排障等核心场景，帮助团队高效管理 Kubernetes 环境。

多集群管理

支持导入多个 Kubernetes 集群，统一进入集群列表与资源视图。

精细权限控制

将不同集群、命名空间的访问权限分配给指定用户，适配多人协作场景。

资源与应用管理

帮助开发人员管理 Kubernetes 集群中运行的应用程序，减少重复、分散的操作。

故障排查入口

在 Web 控制台内查看资源状态、定位应用问题，更快处理集群中的复杂性。

监控指标集成

支持对接 Prometheus、Grafana 等观测能力，帮助团队持续掌握集群运行状态。

轻量部署

支持 Docker 快速启动，也可以通过 1Panel 应用商店快速部署。
